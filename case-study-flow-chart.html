Project Overview

I designed the Admin Feedback Response Flow for a feedback management module within a mental health platform. The system manages feedback collection, viewing, and response workflows for two user types: Clients and Therapists.

The main objective was to create a clear, intuitive flow that allowed administrators to easily view and respond to feedback, while ensuring that both user types had their respective data displayed in an organized and actionable way.

Target Audience

  • Primary: Platform administrators managing feedback.
  • Secondary: Indirectly, Clients and Therapists who provide feedback.

Client Feedback Flow

A new screen auto-loads with pre-selected: Session, Organization, Feedback name, User type = Client

  • Table view lists feedback with therapist name, therapy type, and actions (View).
  • Admin can drill down to see detailed question/answer responses per client.

Therapist Feedback Flow

A new screen auto-loads with pre-selected: Session, Organization, Feedback name, User type = Therapist

  • Table view lists assigned patients, therapy types, and actions (View/Download).
  • Admin can drill down to see detailed responses to specific questions.

Final Outcome

  • Delivered a streamlined feedback flow that reduced admin effort in retrieving and managing responses.
  • Improved data clarity and accessibility through structured tables and filters.
  • Enabled scalability by including CSV exports and pagination.
  • Positive stakeholder feedback for creating a clear separation of flows for clients and therapists, while keeping UI consistent.
